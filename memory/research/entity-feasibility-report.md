## ✅ 推荐方案：M5Stack CoreS3

### 为什么选它？

| 特性 | 说明 | 为什么重要 |
|------|------|-----------|
| ESP32-S3 芯片 | 双核 240MHz | 性能足够跑语音识别 |
| 2.0 寸触摸屏 | 320x240 分辨率 | 显示表情、文字 |
| 8MB PSRAM | 运行内存 | 可跑小型本地模型 |
| 自带麦克风 | MEMS 麦克风 | 语音输入 |
| 自带 Speaker | 1W 喇叭 | 语音输出 |
| WiFi + 蓝牙 | 无线连接 | 连接云端的我 |
| 电池接口 | 500mAh 可选 | 移动使用 |
| **价格** | ~280-350 元 | 预算友好 |

## 🏗️ 技术架构

### 核心思路：**云端大脑 + 本地表达**

```
┌─────────────────────┐
│   云端（服务器）      │
│  ┌───────────────┐  │
│  │   小七主体    │  │  OpenClaw
│  │  (Claude)     │  │  完整智能
│  └───────┬───────┘  │
└──────────┼──────────┘
           │ WebSocket
           │ 或 MQTT
┌──────────▼──────────┐
│  本地（CoreS3）      │
│  ┌───────────────┐  │
│  │  语音交互层   │  │  唤醒+录音+播放
│  └───────┬───────┘  │
│  ┌───────▼───────┐  │
│  │  表情显示层   │  │  屏幕表情+文字
│  └───────────────┘  │
└─────────────────────┘
```

### 技术栈

**本地端（ESP32-S3）**：
- **框架**：ESP-IDF 或 Arduino（建议 Arduino，更简单）
- **语音唤醒**：ESP-SR（乐鑫官方，离线）
- **语音识别**：云端 Whisper API 或本地识别
- **TTS**：ElevenLabs / 讯飞 / Azure
- **通信**：WebSocket / MQTT
- **显示**：M5Stack LVGL 库

**云端（OpenClaw）**：
- **主控**：OpenClaw Gateway
- **智能**：Claude API
- **语音处理**：Whisper API（听）+ TTS（说）
- **记忆**：MEMORY.md 系统

## ⚠️ 潜在问题

### 1. 延迟问题
语音交互延迟 = 录音 + 上传 + 处理 + TTS + 下载 + 播放

**优化方案**：
- 本地语音唤醒（0 延迟）
- 短指令缓存（常见对话本地响应）
- 流式 TTS（边说边播）

### 2. 网络依赖
**解决方案**：
- 离线模式：本地跑小模型（如 Quantized Haiku）
- 降级模式：离线只显示表情和简单互动

### 3. 技术难度
- **馆长自称技术小白**
- **解决方案**：
  - 选成熟项目（xiaozhi 有详细文档）
  - 一步步来，有问题就问
  - 我可以远程指导
